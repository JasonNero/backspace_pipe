# Das Problem

Mehrere Versionen von Python werden auf Windows ohne Python in der PATH Umgebungsvariable durch den py.exe Wrapper ermöglicht. Hat man z.B. Python 2.7 und 3.6 parallel installiert kann man ohne die Python.exe im PATH zu haben in der PowerShell/Cmd die gewünschte Version mit py -<VERSION> benutzen:
> py -2 --version
>>> 2.7.11
> py -3 --version
>>> 3.6

Auch pip lässt sich so nutzen um z.b. virtualenv und den virtualenvwrapper zu installieren:
> py -2 -m pip install virtualenv

ABER... die virtualenvwrapper (egal ob der virtualenvwrapper-win für "cmd" oder der für die PowerShell, der sowieso recht verzwickt zu installieren ist) funktionieren ohne python im PATH nicht.

# Die pragmatischste Lösung:
1. virtualenv installieren:
    py -2 -m pip install virtualenv
2. In den Ordner wechseln, in dem das Venv angelegt werden soll und explizit die gewünschte Version rufen:
    c:\python27\scripts\virtualenv --python=c:\Python27\python.exe meinVenv
3. In das angelegte Venv-Verzeichnis wechseln und das Venv durch "activate" aktivieren:
    Scripts/activate
4. Nun kann mittels pip installiert werden (ohne py-wrapper):
    pip install requests
5. In Maya kann über "site" dieses venv eingebunden werden:
    import site
site.addsitedir("Pfad/zum/venv/meinVenv/Lib/site-packages")
